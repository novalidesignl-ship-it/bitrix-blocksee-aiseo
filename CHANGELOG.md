## [1.11.0] — 2026-05-06

### Добавлено
- **Раздельная генерация «достоинства»/«недостатки»** в кастомных инфоблоках отзывов. Новый чекбокс в настройках кастомного backend'а: «Заполнять Достоинства и Недостатки отдельно от текста». Когда включён:
  - AI-сервер получает флаг `split_pros_cons=true` и отдаёт три раздельных поля: `plusses`, `minuses`, `content` (вместо одного `content`).
  - `CustomBackend::saveForElement` авто-определяет коды свойств для плюсов и минусов в инфоблоке отзывов (кандидаты: `PLUSSES`/`PLUSES`/`PROS`/`PLUS`/`ADVANTAGES`/`GOOD` и `MINUSES`/`CONS`/`MINUS`/`DISADVANTAGES`/`BAD`) и пишет в них значения от AI вместо заглушки `«—»`.
  - Промпт **«недостатков»** настроен мягко: AI выбирает из трёх режимов в зависимости от тона отзыва — отказ-формулировка («не заметил», «нечего сказать»), мизерное замечание («инструкция мелковата»), либо косвенное наблюдение («больше места занимает чем казалось»). Запрещено: «брак»/«не работает»/«обманули» и придумывание дефектов, которых нет в данных товара.
  - Промпт **«плюсов»**: 5–15 слов, по делу, без штампов, без числовых ТТХ.
  - Серверная часть (`BitrixProductReviewsGenerator` на lk.blocksee.ru) расширена ветвлением по флагу `split_pros_cons`. `moderateTone()` применяется ко всем трём полям.
- На стороне модуля: новый геттер `Options::getReviewsCustomSplitProsCons()`, новый ключ `reviews_custom_split_pros_cons` (Y/N, default N). `ApiClient::generateProductReviews` пробрасывает флаг в payload.

### Совместимость
- Старый сервер без поддержки `split_pros_cons` просто игнорирует флаг и возвращает обычный формат — модуль отрабатывает в режиме v1.10.2 (заглушки `«—»` для PLUSSES/MINUSES).
- Чекбокс по умолчанию выключен — поведение существующих установок не меняется.

## [1.10.1] — 2026-05-06

### Исправлено (критично, custom-backend)
- **Авто-заполнение обязательных свойств** в `CustomBackend::saveForElement`. На сайтах вроде tsar-climat.ru в инфоблоке отзывов несколько свойств с `IS_REQUIRED='Y'` (`PLUSSES`/`MINUSES`/`LONG`/`AUTHOR`), не связанных с тем, что отдаёт AI. Битрикс отклонял `CIBlockElement::Add()` без явной ошибки, и юзер видел только «Ни один отзыв не сохранён».
- Backend теперь собирает все required-свойства целевого инфоблока один раз и автозаполняет неуказанные осмысленными заглушками по семантике кода:
  - `AUTHOR`/`AUTHOR_NAME`/`FIO`/`TITLE`/`NAME` → имя автора отзыва.
  - `LONG`/`EXPERIENCE`/`PERIOD`/`USAGE_TIME`/`LONG_TIME` → `«Несколько месяцев»`.
  - `PLUSSES`/`MINUSES`/`PROS`/`CONS` → `«—»` (короткое тире).
  - `COMMENT`/`CONTENT`/`TEXT`/`BODY`/`MESSAGE` → тело отзыва (если контент-target лежит не там).
  - `RATING`/`STARS`/`RATING_VALUE` → оценка отзыва.
  - Прочие S-свойства → `«—»`.
- `E`-свойства (привязки) и `F`-свойства (файлы) не трогаются.
- При неудачном `Add()` теперь логируется `LAST_ERROR`: ответ AJAX вместо «Ни один не сохранён» содержит первые 3 уникальные ошибки от Битрикса (например, «Заполните обязательное поле X»).

## [1.10.0] — 2026-05-06

### Добавлено
- **Custom-backend для отзывов** — новый источник `«Из настроек плагина»`. Покрывает самописные схемы хранения отзывов на Битриксе (например, tsar-climat: на отзыве свойство `PRODUCT` указывает на товар, плюс кастомные коды `RATING`/`AUTHOR`/`PLUSSES`/`MINUSES`).

- Новый класс `Blocksee\Aiseo\Reviews\CustomBackend` (`lib/reviews/custombackend.php`, ~330 строк) реализует интерфейс `Backend` целиком из настроек, без auto-detect'а.
- В `admin/options.php` секция «Кастомный инфоблок отзывов» с дропдауном инфоблоков и текстовыми полями для кодов свойств. Видна всегда, активируется выбором источника `«Из настроек плагина»` сверху.
- В селекторе «Куда сохранять» на страницах `admin/reviews.php` и `admin/reviews_urls.php` появился пункт «Из настроек плагина».
- Затронуто: `lib/options.php` (новые константы и геттеры), `lib/reviews/custombackend.php` (новый), `lib/reviews/factory.php`, `lib/controller/reviews.php`, `include.php` (autoload), `admin/options.php`, `admin/reviews.php`, `admin/reviews_urls.php`.

### Совместимость
- Старые источники (`forum`, `blog`, `iblock` auto-detect, `auto`) работают без изменений. Custom-backend подключается только при явном выборе.

## [1.9.13] — 2026-05-06

### Добавлено
- В поле «Отзывов на товар» (на страницах «Отзывы товаров» и «Отзывы по ссылкам») поддержан **диапазон**: можно ввести `3` (фиксировано) или `1-3` (случайное число в диапазоне для каждого товара). Это выглядит реалистичнее, чем «у всех ровно 3 отзыва» — у одних товаров 1, у других 2-3, как в живых интернет-магазинах.
- Парсинг толерантный: пробелы вокруг дефиса допустимы (`2 - 5`), верхний потолок 20, нижний — 1, max < min автоматически выпрямляется. В confirm-диалоге показывается «по N» либо «от N до M», и оценка общего числа отзывов считается как `total × average(min, max)`.
- Затронуто: `assets/reviews.js`, `assets/reviews_urls.js`, `admin/reviews.php`, `admin/reviews_urls.php`. Серверная часть не менялась — bulk-цикл просто шлёт разные `count` на каждый товар.

## [1.9.11] — 2026-05-06

### Исправлено (критично, bulk-генерация)
- На сайтах с low MySQL `wait_timeout` (60-90 сек на shared-хостингах вроде timeweb) bulk-генерация отзывов и описаний по ссылкам падала с `MySQL Query Error` и **HTML вместо JSON** в AJAX-ответе («Unexpected token 'F', "FILE: /hom"... is not valid JSON»). Корень: AI-вызов длится 30-90 сек, за это время MySQL закрывает idle connection в Битрикс-pool, и следующий SQL `\CIBlockElement::Add` / `SetPropertyValuesEx` упирается в stale connection.
- Добавлен принудительный reconnect MySQL connection **сразу перед записью** в БД в трёх ключевых точках:
  - `ReviewsGenerator::saveReviewsForElement` (через приватный метод `reopenDbConnection`).
  - `Generator::saveDescription`.
  - `CategoryGenerator::saveDescription`.
- Reconnect через `\Bitrix\Main\Application::getInstance()->getConnection()->disconnect()` + `connect()`. Best-effort: если по какой-то причине не получилось — следующий SQL сам попробует и упадёт с понятной ошибкой.

### Исправлено (критично, описания товаров)
- На euro-komplekt и других сайтах с ZeroGPT-ориентированными post-processing-фильтрами `UltimateAISEOGenerator` мог обнулять весь HTML-ответ Sonnet до пустой строки — модуль валился с «All AI models failed». ## [1.9.1] — 2026-05-05

- Пропущенный кейс из v1.8.4 — `personapool.php::filterAlive()`. Ещё один вызов `\CUser::GetList('ID', 'ASC', ...)` с literal strings в by-ref параметрах. ## [1.8.9] — 2026-05-05

### Изменено
- AI-ответы для описаний категорий часто приходят в виде «голых» абзацев + `<h2>` без `<p>`-обёртки. На фронте Aspro Premier (и многих других шаблонах) такой контент склеивается в одну строку — переносы `\n` не превращаются в визуальные разрывы. - Метод `Generator::restoreLatest(int $elementId)` — записывает в товар сохранённый оригинал (последняя версия из истории). Поддерживает все три режима хранения: DETAIL_TEXT, PREVIEW_TEXT, PROPERTY.
- AJAX-action `restoreLatest` в контроллере `Blocksee\Aiseo\Controller\Generator` + кнопка **«↶ Откатить»** в каждой строке таблицы товаров на странице «Описания товаров». Кнопка возвращает текст в textarea и в поле инфоблока.
- Установщик: `installDB()` создаёт таблицу при инсталляции/переустановке. При деинсталляции таблица **не удаляется** — пользовательские бэкапы сохраняются (можно удалить вручную через `\Blocksee\Aiseo\BackupStorage::dropTable()`).

### Архитектура
- Бэкап создаётся ДО `\CIBlockElement::Update` — даже если запись провалится, оригинал в таблице.
- Если оригинал пустой — бэкап не создаётся (нечего хранить).
- На каждый товар + поле максимум 10 версий, чтобы таблица не пухла на больших каталогах.

- `Reviews\Scenarios` — библиотека разнообразных сценариев отзыва (подбираются рандомно при `count > 1`, чтобы тексты не были шаблонными).

- `Options::getIblockTypeId()` + `Options::buildElementEditUrl()` — сборка корректной ссылки на редактор товара для **любого** типа инфоблока (включая `aspro_premier_catalog`, `aspro_next_catalog` и кастомные типы).

- `ReviewsGenerator` переведён на backend-слой — больше не содержит форум-специфичного кода.

### Исправлено

- Фильтр каталогов больше не ограничен `TYPE='catalog'` — подхватывает Aspro Premier (`aspro_premier_catalog`), Next, Allcorp и любые кастомные типы.
