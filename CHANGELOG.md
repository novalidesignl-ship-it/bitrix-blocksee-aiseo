## [1.6.0] — 2026-04-29

### Изменено
- **Убран quality-tier из UI.** Опция «Качество текста» (Стандарт/Высокое) удалена со страницы настроек модуля. Индикатор «Качество» убран из шапок страниц «Описания товаров» и «Генерация по ссылкам». Везде теперь используется одна модель — DeepSeek v4 Pro с дистиллированным журналистским промптом, который раньше был только в high-режиме. На стороне `lk.blocksee.ru` API соответственно: `bsee_models_chain_for_quality()` возвращает одну цепочку независимо от `settings.quality`, `bsee_pick_model_for_quality()` всегда `OPENROUTER_MODEL`, `UltimateAISEOGenerator` всегда работает в режиме «high» (журналистский тон, temp 0.7, без рекламных штампов).
- `Options::getQualityTier()` теперь константно возвращает `'high'` — для обратной совместимости с payload-схемой (поле в payload остаётся, чтобы не ломать API-контракт, но сервер его игнорирует).

### Server-side (lk.blocksee.ru, не в этом репо)
- env: `OPENROUTER_MODEL=deepseek/deepseek-v4-pro`, `OPENROUTER_MODEL_FALLBACK=deepseek/deepseek-v4-flash`. На сбое DS Pro (rate limit, timeout, empty content) автоматический fallback на DS Flash.
- env старые `OPENROUTER_MODEL_HIGH*` сохранены для совместимости, но больше не разделяют поведение (всё ведёт на одну DS Pro цепочку).
- Промпт: всегда журналистско-технический тон без рекламы, без цены и артикула в контексте, лимит на длинное тире и восклицания, post-process чистит pseudo-lists и невалидную HTML-вложенность.

## [1.4.3] — 2026-04-28

### Исправлено
- **Резолв 750+ URL'ов на странице «Генерация по ссылкам»** падал с ошибкой `Unknown column 'DETAIL_PAGE_URL' in 'field list'`. Причина: в `Generator::resolveUrlsAction` Phase 2 (fallback) делала прямой SQL `SELECT DETAIL_PAGE_URL FROM b_iblock_element`, но эта колонка реально не существует — `DETAIL_PAGE_URL` это шаблон на инфоблоке (`b_iblock.DETAIL_PAGE_URL`), который Bitrix подставляет на лету. На небольших партиях все URL'ы матчились в Phase 1 по `CODE`, до Phase 2 не доходило. На больших — хоть один не матчился, Phase 2 запускалась и роняла весь резолв.
- Удалил Phase 2 целиком. Match по символьному коду (`CODE` = последний сегмент URL) корректно резолвит реальные кейсы (`/catalog/.../{element-code}/`). URL без CODE-match помечаются `not_found` (например, если случайно вставили ссылку на раздел, а не товар).

## [1.4.2] — 2026-04-28

### Исправлено
- В модалке «Массовая автоматическая генерация» не отображались категории для выбора. Причина: внутри цикла отрисовки списка товаров переменная `$sections` (содержавшая список категорий инфоблока для модалки) перезаписывалась результатом `bsee_get_sections($elementId)` — категориями конкретного товара. К моменту рендеринга модалки в `$sections` оставались данные от последнего товара на странице, и `foreach` ничего не выводил. Переименовал локальную переменную в `$itemSections`. Теперь все 15 категорий инфоблока корректно отображаются в модалке.

## [1.4.1] — 2026-04-28

- Кнопка **«↶ Откатить выделенные»** в bulk-actions панели страницы «Описания товаров». Откатывает все отмеченные чекбоксами товары к их предыдущей версии описания одним нажатием. ## [1.4.0] — 2026-04-28

### Добавлено
- **Резервные копии описаний.** Перед каждой AI-генерацией старое значение целевого поля (DETAIL_TEXT / PREVIEW_TEXT / свойство) сохраняется в новую таблицу `b_blocksee_aiseo_backups`. Хранится история до 10 последних версий на товар + поле, старые удаляются автоматически.
- Новый класс `Blocksee\Aiseo\BackupStorage` (`lib/backupstorage.php`) — CRUD над таблицей бэкапов: `save`, `getLatest`, `getHistory`, `cleanupOld`, `ensureTable`, `dropTable`.
- Метод `Generator::restoreLatest(int $elementId)` — записывает в товар сохранённый оригинал (последняя версия из истории). Поддерживает все три режима хранения: DETAIL_TEXT, PREVIEW_TEXT, PROPERTY.
- AJAX-action `restoreLatest` в контроллере `Blocksee\Aiseo\Controller\Generator` + кнопка **«↶ Откатить»** в каждой строке таблицы товаров на странице «Описания товаров». Кнопка возвращает текст в textarea и в поле инфоблока.
- Установщик: `installDB()` создаёт таблицу при инсталляции/переустановке. При деинсталляции таблица **не удаляется** — пользовательские бэкапы сохраняются (можно удалить вручную через `\Blocksee\Aiseo\BackupStorage::dropTable()`).

### Архитектура
- Бэкап создаётся ДО `\CIBlockElement::Update` — даже если запись провалится, оригинал в таблице.
- Если оригинал пустой — бэкап не создаётся (нечего хранить).
- На каждый товар + поле максимум 10 версий, чтобы таблица не пухла на больших каталогах.

## [1.3.0] — 2026-04-27

### Добавлено
- Новая админ-страница **«Сервисы → БЛОКСИ: ИИ SEO → Генерация по ссылкам»** (`blocksee_aiseo_list_urls.php`): массовый ввод URL карточек товаров (от 1 до тысяч ссылок), резолв в элементы инфоблока и пакетная AI-генерация описаний.

- CSS-классы `.bsee-item-status.err` и `.bsee-item-status.wait` для отображения статусов ошибки и ожидания в таблицах.

## [1.1.0] — 2026-04-25

### Добавлено
- **Поддержка Aspro Premier** и других решений, где отзывы хранятся не в `forum`, а в модуле `blog` (каталог комментариев). Новый абстрактный слой `Reviews\Backend` + две реализации: `ForumBackend` (по-прежнему для стока/Аспро-Next) и `BlogBackend` (Aspro Premier и подобные).
- `Reviews\Factory::create()` — авто-подбор backend-а по настройке `reviews_source` (`auto` / `forum` / `blog`). В режиме `auto` предпочитается blog, если модуль установлен.
- Установщик `ensureReviewsBlog()` — при установке на сайте с модулем blog автоматически создаёт блог `catalog_comments` (URL настраивается).
- Настройка источника отзывов на странице **«Настройки»**: селект `reviews_source`, поле URL блога (`reviews_blog_url`, по умолчанию `catalog_comments`), видимость разрешённых вариантов зависит от установленных модулей.
- `Reviews\Scenarios` — библиотека разнообразных сценариев отзыва (подбираются рандомно при `count > 1`, чтобы тексты не были шаблонными).
- `Reviews\PersonaPool` — пул персонажей-авторов для Scenarios, смешивание ролей / возраста / стиля речи.
- Жёсткие анти-AI правила в промпте для отзывов: запрет артикулов, технических чисел, стоп-фраз «Купил…», «Рекомендую», «качество на высоте», «оправдал ожидания» и т.п.
- `Options::getIblockTypeId()` + `Options::buildElementEditUrl()` — сборка корректной ссылки на редактор товара для **любого** типа инфоблока (включая `aspro_premier_catalog`, `aspro_next_catalog` и кастомные типы).
- `Options::getCatalogIblocks()` использует `\Bitrix\Catalog\CatalogIblockTable` с фильтром `PRODUCT_IBLOCK_ID=0` — точно различает основной каталог и торговые предложения (SKU), корректно работает в Аспро.

### Изменено
- Модель отзывов: N отзывов генерируются **отдельными** запросами к API (по одному на отзыв) с разными сценариями в `custom_prompt`, а не одним большим батчем — гораздо выше разнообразие.
- `ReviewsGenerator` переведён на backend-слой — больше не содержит форум-специфичного кода.

### Исправлено
- Ссылка на редактирование товара больше не содержит хардкода `type=catalog` — `type` берётся из фактического `IBLOCK_TYPE_ID`.
- Фильтр каталогов больше не ограничен `TYPE='catalog'` — подхватывает Aspro Premier (`aspro_premier_catalog`), Next, Allcorp и любые кастомные типы.

## [1.0.0] — 2026-04-24

### Добавлено
- Модуль `blocksee.aiseo` для 1С-Битрикс 26.x (D7, PHP 8.1+).
- Генерация описаний товаров через защищённый AI-API (`action=generate_product_description`):
  - Админ-страница **«Сервисы → БЛОКСИ: ИИ SEO → Описания товаров»**.
  - Дополнительный промпт, тулбар (инфоблок / категория / статус / поиск).
  - Inline-редактирование, точечная и массовая генерация (курсорная пагинация, сценарии *Заполнить пустые* / *Перезаписать все*).
  - Сохранение в `DETAIL_TEXT` / `PREVIEW_TEXT` / оба / пользовательское свойство.
- Генерация отзывов товаров через защищённый AI-API (`action=generate_product_reviews`):
  - Админ-страница **«Отзывы товаров»**.
  - Автоматически создаёт форум `Отзывы товаров (AI)` при установке.
  - Регистрирует UF-поля `UF_RATING` и `UF_AI_GENERATED` на `FORUM_MESSAGE` — совместимо с Аспро / типовым каталогом.
  - Топик на каждый товар с `XML_ID = iblock_{IBLOCK_ID}_{ELEMENT_ID}`.
  - Случайные даты публикации в настраиваемом диапазоне, fake email по транслитерации имени.
  - Просмотр, редактирование и удаление отзывов прямо из модалки.
  - Массовая генерация с выбором сценария *Только пустые* / *Добавить к имеющимся*.
- Санитайзер текста `TextSanitizer::stripEmoji` — вырезает эмодзи и декоративные символы (SMP pictographs, Misc Symbols, Misc Technical, Geometric Shapes, Misc Symbols and Arrows, Regional Indicators, ZWJ, BOM), схлопывает дублирующиеся пробелы.
- D7-контроллеры: `blocksee:aiseo.generator`, `blocksee:aiseo.reviews` с CSRF-защитой и admin-only доступом.
- Страница настроек с параметрами генерации (temperature, max_tokens, creative_mode), опциями отзывов (кол-во/товар, рейтинг, авто-одобрение, диапазон дат).
- Единый дизайн-система админки на CSS-переменных (синяя палитра).
