# CHANGELOG

Формат: [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/) · Версионирование: [SemVer](https://semver.org/lang/ru/).

## [1.1.0] — 2026-04-25

### Добавлено
- **Поддержка Aspro Premier** и других решений, где отзывы хранятся не в `forum`, а в модуле `blog` (каталог комментариев). Новый абстрактный слой `Reviews\Backend` + две реализации: `ForumBackend` (по-прежнему для стока/Аспро-Next) и `BlogBackend` (Aspro Premier и подобные).
- `Reviews\Factory::create()` — авто-подбор backend-а по настройке `reviews_source` (`auto` / `forum` / `blog`). В режиме `auto` предпочитается blog, если модуль установлен.
- Установщик `ensureReviewsBlog()` — при установке на сайте с модулем blog автоматически создаёт блог `catalog_comments` (URL настраивается).
- Настройка источника отзывов на странице **«Настройки»**: селект `reviews_source`, поле URL блога (`reviews_blog_url`, по умолчанию `catalog_comments`), видимость разрешённых вариантов зависит от установленных модулей.
- `Reviews\Scenarios` — библиотека разнообразных сценариев отзыва (подбираются рандомно при `count > 1`, чтобы тексты не были шаблонными).
- `Reviews\PersonaPool` — пул персонажей-авторов для Scenarios, смешивание ролей / возраста / стиля речи.
- Жёсткие анти-AI правила в промпте для отзывов: запрет артикулов, технических чисел, стоп-фраз «Купил…», «Рекомендую», «качество на высоте», «оправдал ожидания» и т.п.
- `Options::getIblockTypeId()` + `Options::buildElementEditUrl()` — сборка корректной ссылки на редактор товара для **любого** типа инфоблока (включая `aspro_premier_catalog`, `aspro_next_catalog` и кастомные типы).
- `Options::getCatalogIblocks()` использует `\Bitrix\Catalog\CatalogIblockTable` с фильтром `PRODUCT_IBLOCK_ID=0` — точно различает основной каталог и торговые предложения (SKU), корректно работает в Аспро.

### Изменено
- Модель отзывов: N отзывов генерируются **отдельными** запросами к API (по одному на отзыв) с разными сценариями в `custom_prompt`, а не одним большим батчем — гораздо выше разнообразие.
- `ReviewsGenerator` переведён на backend-слой — больше не содержит форум-специфичного кода.

### Исправлено
- Ссылка на редактирование товара больше не содержит хардкода `type=catalog` — `type` берётся из фактического `IBLOCK_TYPE_ID`.
- Фильтр каталогов больше не ограничен `TYPE='catalog'` — подхватывает Aspro Premier (`aspro_premier_catalog`), Next, Allcorp и любые кастомные типы.

## [1.0.0] — 2026-04-24

### Добавлено
- Модуль `blocksee.aiseo` для 1С-Битрикс 26.x (D7, PHP 8.1+).
- Генерация описаний товаров через защищённый AI-API (`action=generate_product_description`):
  - Админ-страница **«Сервисы → БЛОКСИ: ИИ SEO → Описания товаров»**.
  - Дополнительный промпт, тулбар (инфоблок / категория / статус / поиск).
  - Inline-редактирование, точечная и массовая генерация (курсорная пагинация, сценарии *Заполнить пустые* / *Перезаписать все*).
  - Сохранение в `DETAIL_TEXT` / `PREVIEW_TEXT` / оба / пользовательское свойство.
- Генерация отзывов товаров через защищённый AI-API (`action=generate_product_reviews`):
  - Админ-страница **«Отзывы товаров»**.
  - Автоматически создаёт форум `Отзывы товаров (AI)` при установке.
  - Регистрирует UF-поля `UF_RATING` и `UF_AI_GENERATED` на `FORUM_MESSAGE` — совместимо с Аспро / типовым каталогом.
  - Топик на каждый товар с `XML_ID = iblock_{IBLOCK_ID}_{ELEMENT_ID}`.
  - Случайные даты публикации в настраиваемом диапазоне, fake email по транслитерации имени.
  - Просмотр, редактирование и удаление отзывов прямо из модалки.
  - Массовая генерация с выбором сценария *Только пустые* / *Добавить к имеющимся*.
- Санитайзер текста `TextSanitizer::stripEmoji` — вырезает эмодзи и декоративные символы (SMP pictographs, Misc Symbols, Misc Technical, Geometric Shapes, Misc Symbols and Arrows, Regional Indicators, ZWJ, BOM), схлопывает дублирующиеся пробелы.
- D7-контроллеры: `blocksee:aiseo.generator`, `blocksee:aiseo.reviews` с CSRF-защитой и admin-only доступом.
- Страница настроек с параметрами генерации (temperature, max_tokens, creative_mode), опциями отзывов (кол-во/товар, рейтинг, авто-одобрение, диапазон дат).
- Единый дизайн-система админки на CSS-переменных (синяя палитра).
